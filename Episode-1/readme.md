-->> Waterfall model overview
The Waterfall Model is one of the oldest Software Development Life Cycle (SDLC) approaches where development flows step-by-step in a fixed sequence.

1. Requirement Phase
 Work:
Gather and analyze all project requirements
Define functional + non-functional requirements
Create SRS document
 Roles:
Business Analysts
Project Managers
Stakeholders
Product Owners


 2. Design Phase
 Work:
Convert requirements into system design
 Types:
High-Level Design (HLD) → Architecture
Low-Level Design (LLD) → Component details
 Roles:
Solution Architects
UX/UI Designers
System Designers
Technical Leads


 3. Development Phase
 Work:
Coding based on design
Build features and integrate modules
 Roles:
Software Developers
Frontend / Backend Developers
Database Administrators
DevOps Engineers


 4. Testing Phase
 Work:
Find and fix bugs
 Types:
Unit Testing
Integration Testing
System Testing
Acceptance Testing
 Roles:
QA Engineers
Testers
Test Leads
Automation Engineers


 5. Deployment Phase
 Work:
Release software to production
Configure system + train users
 Roles:
DevOps Engineers
System Administrators
Release Managers
IT Support


 6. Maintenance Phase
 Work
Fix bugs after release
Add updates and improvements
 Roles:
Support Engineers
Maintenance Team
Developers
Customer Support
 Key Insight (VERY IMPORTANT )

 In Waterfall Model:

Note: Each phase is completed 100% before moving ahead
No backtracking easily
Heavy documentation at every stage
